# Инструкция по деплою на GitHub Pages

## Шаг 1: Создать репозиторий на GitHub

1. Зайди на [github.com](https://github.com)
2. Нажми кнопку "+" в правом верхнем углу → "New repository"
3. Название репозитория: `LendVik` (или любое другое)
4. Выбери **Public** (для бесплатного GitHub Pages)
5. **НЕ** создавай README, .gitignore или лицензию (у нас уже есть)
6. Нажми "Create repository"

## Шаг 2: Подключить локальный репозиторий к GitHub

После создания репозитория GitHub покажет инструкции. Выполни команды:

```bash
# Если репозиторий называется LendVik и твой username, например, "smoln"
git remote add origin https://github.com/твой-username/LendVik.git
git branch -M master
git push -u origin master
```

**Важно:** Замени `твой-username` на свой реальный username на GitHub!

## Шаг 3: Настроить GitHub Pages

1. Зайди в свой репозиторий на GitHub
2. Нажми на вкладку **Settings** (вверху справа)
3. В левом меню найди **Pages**
4. В разделе "Source":
   - Выбери **Deploy from a branch**
   - Branch: выбери **master** (или **main**, если у тебя main)
   - Folder: выбери **/ (root)**
5. Нажми **Save**

## Шаг 4: Получить ссылку на сайт

Через 1-2 минуты сайт будет доступен по адресу:

```
https://твой-username.github.io/LendVik/
```

Например, если твой username `smoln`, то:
```
https://smoln.github.io/LendVik/
```

## Обновление сайта

После любых изменений просто делай:

```bash
git add .
git commit -m "Описание изменений"
git push
```

Сайт обновится автоматически через 1-2 минуты.

## Альтернатива: Netlify (если GitHub Pages не подходит)

1. Зайди на [netlify.com](https://netlify.com)
2. Зарегистрируйся (можно через GitHub)
3. Нажми "Add new site" → "Import an existing project"
4. Выбери GitHub и свой репозиторий
5. Настройки:
   - Build command: оставь пустым
   - Publish directory: `/`
6. Нажми "Deploy site"

Сайт будет доступен сразу по адресу типа: `lendvik-123.netlify.app`
